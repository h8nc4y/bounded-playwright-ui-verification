# Support

Use GitHub issues for support requests that belong in the public repository.

## Good Support Requests

Include:

- The agent runtime or tool where you installed the skill.
- The Web UI stack being verified.
- The browser automation tool used.
- The bounded server command shape.
- The exact check that is blocked or unclear.
- Any relevant `未確認` categories.

Do not include secrets, auth cookies, private logs, customer data, or screenshots
that reveal private systems.

## Where To File

- Use the bug report template when the skill or validation scripts behave
  incorrectly.
- Use the feature request template for bounded verification improvements.
- Use a normal discussion or issue only when the templates do not fit.

Security-sensitive reports should follow [SECURITY.md](SECURITY.md).
